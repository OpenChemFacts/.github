## ğŸ“‹ Overall Process

### 1. Search for / Open an Issue

- Check whether a similar issue already exists
- If you propose a new issue, first open a request issue

### 2. Wait for Initial Feedback

- A maintainer (currently: @albanfournier) will validate the idea, ask questions, or decline it

### 3. Fork the Relevant Repository

Fork the repository you want to contribute to.

### 4. Create a Branch

Use a descriptive branch name following these conventions:

- **For a feature:** `feat/feature-name`
- **For a bug fix:** `fix/bug-name`

### 5. Code and Add Tests

- Write clean, well-documented code
- Add tests for new functionality
- Ensure existing tests pass
- *To be completed with more specific guidelines*

### 6. Open a Pull Request (PR)

When opening a PR, make sure to:

- Describe what the PR does
- Link the corresponding issue (e.g., `Closes #123` or `Fixes #456`)
- Provide context and rationale for the changes

### 7. Review & Merge

- A maintainer reviews the PR and may request changes
- Address any feedback or requested changes
- Once approved, the PR is merged
